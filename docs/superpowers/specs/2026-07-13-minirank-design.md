# MiniRank 설계 문서

## 개요
공통수학1 '행렬' 단원 발표 시연용 인터랙티브 웹 데모. 가상 미니 인터넷(5~8개 페이지)의
링크 구조를 행렬로 표현하고, PageRank 멱승법 반복 계산 과정을 실시간 시각화한다.
행렬 곱셈 → 알고리즘 → 순위 변동의 흐름을 라이브로 증명하는 것이 목적.

## 기술 스택 결정
- **파일 구조**: `index.html` 단일 파일. 인라인 `<style>` + `<script>`. 외부 요청 0개.
- **외부 라이브러리**: 없음 (D3.js, Chart.js 모두 미사용). 오프라인 완전동작이 PRD 성공기준
  필수 항목이라 CDN 의존을 배제. 그래프/화살표/차트 전부 Canvas 2D 직접 구현.
- **렌더링**: Canvas 2D — 원형 노드(그라데이션+글로우), 화살표 링크, 드래그, hover 판정은
  좌표 거리 계산으로 직접 구현.
- **수렴 차트**: 같은 Canvas 기법으로 라인 차트 직접 그리기 (반복횟수 x축, 랭크값 y축).
- **노드 이름 테마**: 웹사이트 테마. 기본 6개 — 네이버, 학교 홈페이지, 내 블로그, 유튜브,
  위키백과, 인스타그램. 10개까지 확장 시 후보: 디시인사이드, 넷플릭스.
- **배포**: 로컬 파일 더블클릭 실행이 1순위. git 커밋으로 이력 관리.

## 데이터 모델
```js
node: { id, name, x, y, dragging, rank, prevRank }
link: { source: nodeId, target: nodeId }  // directed, 중복 없음, self-loop 없음
state: {
  nodes: [], links: [],
  damping: 0.85,
  iteration: 0,
  rankHistory: [],       // [{iter, ranks: {id: value}}, ...] — 수렴차트용
  mode: 'idle' | 'auto',
  converged: false,
  highlighted: { type: 'node'|'link'|null, id },  // 그래프↔행렬 양방향 hover 연동용 단일 소스
  matrixView: 'adjacency' | 'transition',
  presentationMode: false,
}
```

## 수학 로직 (PRD 5장 기준)
- **인접행렬 A**: 링크 j→i 존재 시 `A[i][j] = 1`, 아니면 0. (열=출발 페이지, 행=도착 페이지)
- **전이행렬 M**: 페이지 j가 아웃링크 k개 보유 시, j가 링크하는 각 i에 대해 `M[i][j] = 1/k`.
  아웃링크 0개(dangling node)인 열은 전체를 `1/n`으로 채움 (모든 페이지에 고르게 분배).
- **반복식**: `r(t+1)[i] = d * Σ_j(M[i][j] * r(t)[j]) + (1-d)/n`
- **초기값**: `r(0) = [1/n, 1/n, ..., 1/n]`
- **수렴 판정**: `max(|r(t+1)[i] - r(t)[i]|) < 0.0001` 이거나 반복 100회 도달 시 정지.

## 컴포넌트 설계

### 1. 그래프 에디터 (Canvas, 좌측 패널)
- 노드 원형 자동배치(초기), 이후 자유 드래그로 위치 이동·저장.
- 노드 클릭 A → 클릭 B: A→B 방향 링크 생성 (링크 추가 모드 토글 버튼으로 진입).
- 화살표 클릭: 해당 링크 삭제 (클릭 좌표-선분 거리 판정).
- 노드 추가/삭제 버튼: 3~10개 범위 제한, 버튼 disable로 경계 표시.
- 노드 크기 = rank 값에 비례(최소/최대 반지름 clamp), requestAnimationFrame으로 부드러운 보간.
- 노드 색상 = 순위 그라데이션 (1위 강조색, 그 외 그라데이션 스케일).

### 2. 행렬 패널 (우측 상단)
- 인접행렬/전이행렬 토글 버튼.
- 그래프 상태 변경 시 즉시 재계산·재렌더 (DOM 테이블, 셀 단위 hover 이벤트).
- `state.highlighted` 읽어서: 노드 hover → 해당 행/열 하이라이트, 셀 hover → 대응 그래프
  노드/화살표 하이라이트. 그래프 캔버스도 같은 state 구독.

### 3. PageRank 계산 엔진
- `stepOnce()`: 행렬-벡터 곱 1회 수행, rankHistory에 push, iteration++, 수렴 판정.
- "다음 단계" 버튼: stepOnce() 1회 호출.
- "계산 시작"(자동실행): setInterval로 stepOnce() 반복 호출 (애니메이션 간격 ~400ms), 수렴 시 자동 정지.
- 감쇠계수 d 슬라이더(0.5~1.0, 기본 0.85): 값 변경 시 현재 계산 리셋 후 r(0)부터 재시작.

### 4. 시각화
- 노드 크기/색상: 위 그래프 에디터 항목에 통합.
- 순위표 패널(우측 하단): 페이지명+랭크값+순위, 매 스텝마다 순위 변동 시 행 위치 애니메이션(transition).
- 수렴 차트(하단 전체 폭): rankHistory 기반 Canvas 라인차트, 페이지별 색상 = 그래프 노드 색상과 통일.

### 5. 프리셋 시나리오
- **A 균형**: 6노드, 사이클 + 교차 링크 2~3개 추가한 자연스러운 웹 구조.
- **B 허브**: 5개 페이지가 모두 특정 1개 페이지로 링크 (백링크 SEO 시연용).
- **C 스팸팜**: 3개 노드가 서로만 링크하는 고립 그룹 + 나머지 정상 페이지 3개. damping 낮춰서 완화 효과 시연.
- **초기화 버튼**: 프리셋 A로 리셋 + 계산 상태 초기화.
- 프리셋 전환 시 그래프/행렬/순위표/차트 전부 즉시 리셋.

### 6. 발표 모드
- 폰트/노드 확대 토글: CSS 클래스 스위치로 전역 스케일 조정.
- 다크 모드: CSS 변수 스위치.
- 키보드 단축키: `Space`=다음 단계, `R`=초기화, `A`=자동실행 토글. (입력 포커스 중엔 비활성화)

## Out of Scope
실제 웹 크롤링, 서버/DB, 모바일 대응, 로그인/저장 기능. (PRD 3.7)

## 성공 기준 (PRD 8장)
- 링크 추가/삭제 → 행렬 갱신 → 재계산 → 순위 변동 10초 내 시연 가능
- 행렬 성분↔그래프 화살표 hover 즉시 대응
- 단계별 실행에서 "행렬 곱 1회 = 랭크 1회 갱신" 시각적으로 명확
- 빔프로젝터 뒷자리에서 노드 이름/숫자 판독 가능 (발표 모드 확대 토글로 보강)
- 인터넷 없이 로컬 파일로 완전 동작 (외부 의존 0개로 충족)
